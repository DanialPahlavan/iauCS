Chapter 1: Getting to Know HTML and CSS

This chapter provides a foundational understanding of HTML and CSS, the core technologies for building web pages.

The Basic Web Development Process

Creating a web page follows a simple, repetitive cycle:

  1. Edit: Write or change your code in a text file.
  2. Save: Save the file with an .html or .htm extension.
  3. Reload: Refresh the page in your web browser to see the changes.

You can use any simple text editor, like Notepad (Windows) or TextEdit (Mac).

Getting to Know HTML (HyperText Markup Language)

HTML provides the structure for your web page content.

Key Concepts:

  • Elements: The basic building blocks of HTML. They define the structure and content of a page.
  • Tags: Used to mark the beginning and end of an element.
    • Start Tag: <element>
    • End Tag: </element> (note the slash /)
    • Example: <em>important</em> makes the word "important" italic.
  • Attributes: Provide additional information about an element and are placed inside the start tag.
    • Syntax: <tag attribute="value">
    • Example: <a href="page2.html">Go to Page 2</a>, where href is the attribute.

A Barebones HTML Page

Every HTML page should have this basic structure:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="utf-8"> <title>Page Title Goes Here</title> </head> <body> <!-- Your visible content goes here --> Hello HTML World! </body> </html>
  • <!DOCTYPE html>: Declares the document type. Always the first line.
  • <html>: The root element that wraps everything. lang="en" specifies the language is English.
  • <head>: Contains meta-information about the page (not visible on the page itself).
  • <meta charset="utf-8">: Specifies the character encoding.
  • <title>: Sets the text that appears in the browser tab.
  • <body>: Contains all the content that is visible to the user, like text, images, and links.

Getting Familiar with CSS (Cascading Style Sheets)

CSS is used to style and control the visual presentation of your web page (colors, fonts, layout).

Key Concepts:

  • Style Rule: A block of CSS code that applies styles to selected elements. It consists of a selector and a declaration block.
  • Selector: The part of the rule that targets the HTML element(s) you want to style. Example: h1 targets all <h1> elements.
  • Declaration Block: The part inside the curly braces {}.
  • Declaration: A single style instruction, made of a property and a value, ending with a semicolon.
    • Syntax: property: value;
    • Example: color: blue; or font-size: 72px;

Example of a CSS Rule:

h1 { font-size: 72px; text-align: center; color: blue; }

Three Ways to Apply CSS

1. Inline Style

A style attribute is added directly to an HTML tag. Good for single, small changes.

<h1 style="color: red;">A Red Heading</h1>

2. Internal Style Sheet

CSS rules are placed inside a <style> tag within the HTML page's <head> section. Useful for styling a single page.

<head> <style> h1 { border-style: solid; } </style> </head>

3. External Style Sheet (Best Practice)

All CSS rules are in a separate file (e.g., styles.css). This file is linked to any HTML page that needs the styles, which is great for site-wide consistency.

In the HTML file's <head>:

<link rel="stylesheet" href="styles.css">
