HTML Cheatsheet

This cheatsheet provides a quick reference for common HTML tags, their purposes, and whether they are specific to HTML5 or available in earlier versions (e.g., HTML4 or prior).

Document Structure

Tag Description HTML5?
<!DOCTYPE html> Declares the document type as HTML5. Must be the first line of the document. Yes
<html> Root element of an HTML page, contains all other elements. No
<head> Contains metadata, links to stylesheets, scripts, and other resources. No
<body> Contains the visible content of the webpage, such as text, images, and media. No
<meta> Defines metadata like character set, viewport settings, or page description. No*
<title> Sets the title of the webpage, displayed in the browser tab. No
<link> Links external resources like CSS files to the HTML document. No
<script> Embeds or links to JavaScript code. No
<style> Embeds CSS styles directly in the HTML document. No

*Note: <meta> is not new to HTML5, but HTML5 introduced new attributes like charset and viewport settings.

Headings

Tag Description HTML5?
<h1> to <h6> Defines headings, with <h1> being the largest/most important and <h6> the smallest/least important. No

Text Formatting

Tag Description HTML5?
<p> Defines a paragraph of text. No
<strong> Indicates strong importance, typically displayed as bold text. No
<em> Indicates emphasized text, typically displayed as italicized text. No
<b> Displays text in bold (presentational, use <strong> for semantic emphasis). No
<i> Displays text in italics (presentational, use <em> for semantic emphasis). No
<u> Underlines text (use with caution, as it may confuse users with links). No*
<mark> Highlights text for reference or emphasis. Yes
<small> Displays smaller text, often for fine print or disclaimers. No*
<del> Indicates deleted text, typically displayed with a strikethrough. No
<ins> Indicates inserted text, typically underlined. No
<sub> Defines subscript text, e.g., for chemical formulas (H₂O). No
<sup> Defines superscript text, e.g., for exponents (x²). No

*Note: <u> and <small> were deprecated in HTML4 but redefined with semantic meaning in HTML5.

Links and Navigation

Tag Description HTML5?
<a> Creates a hyperlink to another page, file, or anchor. Uses href attribute for the destination. No*
<nav> Defines a navigation section, typically for menus or links. Yes

*Note: <a> is not new, but HTML5 added attributes like download and ping.

Lists

Tag Description HTML5?
<ul> Creates an unordered (bulleted) list. No
<ol> Creates an ordered (numbered) list. No*
<li> Defines a list item within <ul> or <ol>. No
<dl> Defines a description list. No
<dt> Defines a term in a description list. No
<dd> Provides the description for a term in a description list. No

*Note: <ol> supports new HTML5 attributes like reversed.

Images and Media

Tag Description HTML5?
<img> Embeds an image. Requires src for the image path and alt for accessibility. No*
<audio> Embeds audio content with controls for playback. Yes
<video> Embeds video content with controls for playback. Yes
<source> Specifies media sources for <audio> or <video> elements. Yes
<figure> Groups media (e.g., images) with a caption. Yes
<figcaption> Defines a caption for a <figure> element. Yes

*Note: <img> is not new, but HTML5 added attributes like srcset for responsive images.

Tables

Tag Description HTML5?
<table> Defines a table to display data in rows and columns. No
<tr> Defines a table row. No
<th> Defines a table header cell, typically bold and centered. No
<td> Defines a table data cell. No
<thead> Groups header content in a table. No
<tbody> Groups body content in a table. No
<tfoot> Groups footer content in a table. No
<caption> Defines a table caption, typically displayed above the table. No

Forms

Tag Description HTML5?
<form> Creates a form for user input, with attributes like action and method. No*
<input> Defines an input field (e.g., text, checkbox, radio, submit). No*
<textarea> Creates a multi-line text input area. No
<button> Defines a clickable button, often used for form submission. No
<select> Creates a dropdown menu. No
<option> Defines an option within a <select> element. No
<label> Associates a label with a form control, improving accessibility. No
<fieldset> Groups related form elements together. No
<legend> Provides a caption for a <fieldset>. No

*Note: <form> and <input> support new HTML5 attributes and input types (e.g., email, date, range).

Semantic Elements

Tag Description HTML5?
<header> Defines a header section, often containing logos, titles, or navigation. Yes
<footer> Defines a footer section, often containing contact info or copyright. Yes
<main> Represents the main content of the document. Yes
<article> Defines independent, self-contained content, like a blog post. Yes
<section> Defines a thematic grouping of content. Yes
<aside> Defines content indirectly related to the main content, like sidebars. Yes
<details> Creates a disclosure widget that can be toggled to show/hide content. Yes
<summary> Defines a summary or heading for a <details> element. Yes

Inline and Block Elements

Tag Description HTML5?
<div> A block-level container for grouping elements. No
<span> An inline container for styling or scripting small portions of text. No

Embedded Content

Tag Description HTML5?
<iframe> Embeds another HTML page or content within the current page. No*
<embed> Embeds external content, like multimedia, at a specified point. No*
<object> Embeds external resources, such as PDFs or plugins. No*
<canvas> Provides a space for drawing graphics via JavaScript. Yes
<svg> Embeds scalable vector graphics directly in HTML. Yes

*Note: <iframe>, <embed>, and <object> are not new, but HTML5 enhanced their functionality (e.g., <iframe> supports sandbox).

Metadata and Scripting

Tag Description HTML5?
<base> Specifies the base URL for all relative URLs in the document. No
<noscript> Defines content to display if JavaScript is disabled. No

Other Tags

Tag Description HTML5?
<br> Inserts a line break. No
<hr> Creates a horizontal rule (line) to separate content. No*
<abbr> Defines an abbreviation or acronym, with optional title attribute. No
<address> Indicates contact information for a person or organization. No
<cite> Defines the title of a work, like a book or movie. No*
<blockquote> Indicates a long quotation, often indented. No
<q> Defines a short inline quotation. No
<code> Displays inline code snippets. No
<pre> Displays preformatted text, preserving whitespace and line breaks. No

*Note: <hr> and <cite> have redefined semantic meanings in HTML5.

Deprecated Tags (Avoid in Modern HTML)

Tag Description HTML5?
<font> Deprecated; use CSS for font styling. No
<center> Deprecated; use CSS for centering content. No
<strike> Deprecated; use <del> or CSS for strikethrough. No

Notes

  • Attributes: Most tags support global attributes like id, class, style, title, and event handlers (e.g., onclick). HTML5 introduced new attributes like data-*, hidden, and required.
  • Semantic HTML: HTML5 emphasizes semantic tags (<article>, <section>, etc.) for better accessibility and SEO.
  • Accessibility: Always include alt attributes for <img> and associate <label> with form controls.
  • HTML5-Specific Features: HTML5 introduced new input types (e.g., email, date), APIs (e.g., Geolocation, Web Storage), and improved multimedia support.
